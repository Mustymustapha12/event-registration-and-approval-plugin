DISI Summit Portal
Version: 0.5.4

This plugin manages DISI Summit 2026 registrations inside WordPress.

Main features:
- Captures registrations from a selected form provider, selected registration form, and optional selected group booking form.
- Supports installed providers when available: Fluent Forms, Forminator, Contact Form 7, Gravity Forms, and WPForms.
- Stores registrant name, email, phone, registration type, source form, source entry ID, submitted form data, status, approval details, rejection reason, and payment amounts.
- Registration types supported for pricing are Professional, VIP, Academic/Researcher, Student, and Group Booking.
- Group Booking Form entries are automatically captured as Group Booking registrations.
- Group Booking tickets show the detected number of admitted people instead of Admit One.
- Group Booking tickets include relevant submitted group booking fields for verification context, with attendee names title-cased and shown on separate lines where possible.
- Backup license keys are available for owner fallback approval.
- Sponsorship enquiries can be captured from a configured sponsorship form and viewed from their own menu with email and call actions.
- Duplicate registration submissions blocked by matching email or phone are stored in a Duplicate Entries queue for guarded approval or rejection.
- Group Booking pricing is calculated from the configured group amount per person multiplied by the detected number of group participants.
- Workshop Payment Amount and Exhibition Payment Amount are treated as add-ons to the selected registration type amount for subsequent usage.
- Total Amount to Pay is calculated as registration type amount plus workshop and exhibition add-on amounts when selected in the submitted form.
- Admins can approve registrations from the registration detail page.
- Approval emails notify recipients that their DISI Summit 2026 registration has been approved and include registration number, registration type, registration amount, workshop add-on amount, exhibition add-on amount, total amount, Paystack payment button, and fallback Paystack link.
- Admins can reject registrations from the registration detail page and must provide a rejection reason.
- Rejection emails include the DISI Summit 2026 branding, registration number, and rejection reason.
- Registration details hide internal form metadata fields such as Fluent Form embedded post ID, Fluentform nonce, and WordPress HTTP referer.
- Registrations can be removed automatically when supported source form entries are deleted.
- Integration amount fields display thousands separators while remaining usable for pricing calculations.
- The Integration page includes a VIP Amount field for VIP registration pricing.
- The Integration page includes an Exhibition Payment Amount field that behaves like the Workshop add-on field.
- Database schema upgrades run automatically after in-place plugin updates.
- Portal-side delete controls are removed. Registrations are deleted from the portal when their supported source form entry is deleted.
- Paystack test/live API settings include secret key, public key, and callback/thank-you URL.
- Approval initializes a unique Paystack transaction and emails its authorization URL to the participant.
- Paystack references and authorization URLs are stored against registrations.
- Payment callbacks are verified against Paystack, including reference, amount, currency, email, mode, and registration metadata.
- Successfully verified registrations are marked paid and include payment time and Paystack transaction ID.
- Workshop-only registrations are priced using only the configured workshop amount.
- VIP registrations are detected from supported form submissions and priced from the configured VIP amount.
- Group bookings support a custom registration amount during approval, with any workshop and exhibition add-ons applied separately.
- Fluent Forms trash, single deletion, and bulk permanent deletion events remove matching portal registrations.
- Registration lists can be filtered by paid or unpaid payment status.
- Payment status is displayed with clear badges and amounts use the naira symbol.
- Approved registrations can have their payment-link email resent from the detail page.
- Unpaid Paystack verification results now report the transaction status directly.
- The dashboard shows total, pending, approved, rejected, and paid registration counts.
- The dashboard shows accumulated registration fees, workshop add-ons, exhibition add-ons, total expected amount, and verified amount paid.
- The resend payment email action is available only for approved registrations that are still unpaid.
- Registration exports use the active type, approval status, payment status, and search filters.
- CSV exports include stored registration, source, approval, payment, and all submitted form fields.
- PDF exports include a complete readable detail section for every matching registration.
- PDF exports use a polished landscape table so multiple registrations fit on each page.
- PDF pages include the DISI logo, branded report banner, export filters, financial summary, repeating table headings, alternating rows, and page numbers.
- Each PDF table row includes registration identity, contact, type, approval status, payment status, amount, submission date, source data, payment reference, fee breakdown, and every submitted form field.
- Dashboard amount cards use a smaller type size and one decimal place for large totals.
- Active registration, workshop, exhibition, and expected totals exclude rejected registrations.
- Rejected registration value is displayed in a separate dashboard card.
- Forminator generated field IDs are mapped to their configured human-readable labels when possible.
- Generated provider field names such as name-1, email-1, and phone-1 are cleaned for display and export without changing stored integration behavior.
- Group bookings have no registration or total amount until an administrator enters the custom amount during approval.
- Existing pending group booking amounts are reset safely during the database upgrade.
- Group booking approval requires a positive custom amount and formats thousands separators while typing.
- Pending group booking lists and details show that the amount will be set during approval.
- The dashboard includes an Approved but Unpaid registration card and renames Pending Registrations to Pending Approvals.
- Dashboard amount cards use responsive sizing and overflow protection for multi-million-naira values.
- Forminator single and bulk entry deletion hooks remove the matching portal registrations.
- Approved registrations with verified paid status receive a secure E-ticket automatically by email.
- E-ticket emails include a branded PDF attachment, a secure electronic-ticket link, participant details, and a QR code.
- The QR code directs event staff to the participant's secure electronic ticket.
- Printable PDF tickets use a medium landscape boarding-pass layout with a participant section, perforated divider, and QR verification stub.
- The E-ticketing page can be filtered by registration type, ticket issuance, scan status, and participant search.
- Registration and E-ticketing filters include VIP as a registration type.
- E-ticketing lists support 10, 25, or 50 participants per page with pagination and matching record totals.
- Filtered E-ticketing records can be exported as CSV or a branded landscape PDF report.
- Each WordPress installation now requires a site-specific owner approval key before portal operations are enabled.
- The plugin uses a custom manage_disi_portal capability so role editor plugins can grant DISI Portal access to non-administrator roles.
- License approval is tied to the normalized WordPress site URL and a unique local installation ID.
- License verification is performed locally with a signed key and does not rely on a subscription server, recurring network request, or expiration date.
- Unapproved installations retain access only to the License page while registration capture, Paystack, exports, E-ticketing, and public ticket endpoints remain inactive.
- The private license signing key and owner approval tool are not included in the installable plugin ZIP.

Owner approval workflow:
1. The site administrator opens DISI Portal > License and sends the displayed request code to the plugin owner.
2. The plugin owner runs: php owner-tools/generate-license.php 'DISI-REQ-...'
3. The generated DISI-LIC activation key is pasted into the site's License page.
4. Once approved, normal plugin behavior continues without remote license checks or expiry interruptions.
- The E-ticketing admin menu lists all approved and paid participants with ticket issue time, email delivery time, scan count, and latest scan time.
- Admins can issue missing tickets, resend ticket emails, and preview electronic tickets without increasing the scan count.
- Opening a valid participant ticket records the scan count and latest scan time for admin tracking.
- Ticket URLs use random private tokens and invalid or ineligible tickets return a not-found page.
- CSV exports include ticket number, electronic-ticket URL, issue and email times, scan count, and latest scan time.
- PDF registration exports include ticket number and scan history when a ticket has been issued.
- CSV values are protected against spreadsheet formula execution.
- PDF generation is bundled through FPDF 1.86 and does not require a server PDF extension.
- The plugin applies DISI branding colors:
  Dark Azure #172b3b
  Dark Turquoise #157664
  Gray Green #53965c
  Yellow Green #a5b73c
  Yellow #ffc801
- DISI logo assets are bundled in assets/images.

Upload behavior:
- The plugin package uses a stable internal plugin folder so uploading newer versions replaces the previously installed DISI Summit Portal instead of installing a separate plugin.
